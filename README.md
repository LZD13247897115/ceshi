实现方式：

不进行数据割接，华为侧负责前端页面实现，电签侧提供接口调用，接口需要增加鉴权逻辑，采用前后端分离方式完成迁移。


迁移要求：

菜单功能范围保持不变，主要完成菜单迁移和拆分前后端实现。

迁移之后的使用人员通过BOSS工号访问ASP系统，访问需要金库审批，跟ASP系统配置菜单级金库，涉敏字段需要模糊化处理，根据工号令牌权限配置不同菜单操作

迁移后角色和令牌不进行全量迁移，上线前配置部分用户和管理员权限，并在电签平台进行引导迁移新平台试用和申请权限，后续用户通过地市管理员进行按需配置。

存在同一个界面，不同权限多个菜单的情况比如【工单查询(员工)、工单查询(市)、工单查询(渠道)、工单查询(区)、工单查询】，迁移后，迁移到一个页面，通过工号令牌进行控制

菜单涉及新增考虑分页查询接口，考虑安全性，前后端分离时接口增加鉴权逻辑，防止别人通过curl直接访问

接口调用次数、地市、工号访问记录等信息需要埋点，能够进行汇总统计

工单补录菜单增加字典表控制，可以通过配置屏蔽指定类型工单补录功能。

2.1.9.8.1.1原始通信监听
原始通信内容：是指通信双方（只要其中一方涉及自然人）之间的实际通信内容，包括语音类、短信/彩信类、传真类、数据业务（如即时消息、Email、视频通信、网页浏览等）类等。
2.1.9.8.1.1.1红线要求
RL-10.1.1-1	对于要求不能提供合法监听接口的市场（如英国）:
必须在提供给当地监管机构/客户的源代码中彻底删除合法监听接口特性，提供干净版本。
RL-10.1.1-2	对于支持不同合法监听接口标准（如欧洲ETSI,美国的ANSI，俄罗斯的SOSM及中国的相关标准）的产品:
必须通过版本的形式隔离（如编译宏、模块化加载，不允许采用License控制），确保版本中只存在符合当地合法监听接口标准的代码。
RL-10.1.1-3	如需支持合法监听接口标准之外的非标准需求或不当使用可能侵犯用户通信自由/隐私的特性:
必须通过版本的形式隔离（如编译宏、模块化加载，不允许采用License控制），确保特定地区的非标准需求只在当地销售的版本中存在，其销售或提供应当遵守所适用的法律法规。
RL-10.1.1-4	合法监听接口相关功能必须在产品资料中受限公开，产品资料严格按国家/地区区分版本，即:发往A国家的版本只能有对应A国家标准的相关资料。
合法监听接口的所有在CPI提及的具体资料必须与产品的其他资料隔离，不向客户的所有人员公开，仅向客户/政府特定人员公开。
受限公开的合法监听接口相关资料应在产品CPI资料中提及。参考如下描述：
“合法监听接口特性受客户与华为签订的合同约束，可能不在您的购买或使用范围之内。该特性相关文档属机密信息，不在本文档包中提供。”
RL-10.1.1-5	必须受限公开的合法监听接口相关资料包括售前资料、售后资料、X接口定义。对于禁止华为提供合法监听接口功能的地区（如英国），系统部在向客户提供产品售前资料时应删除合法监听相关内容；对购买了合法监听接口特性的客户，系统部应主动向其合法监听相关部门提供符合该国技术要求的合法监听接口售后资料。
RL-10.1.2-1	系统中和合法监听相关的接口禁止存在合法监听网关正常接口之外的任何其它调用方式，包括命令、脚本、调试接口等。
RL-10.1.2-2	具备合法监听接口的产品/系统需防止被监听对象以及任何未经授权的第三方感知，防止监听敏感信息或内容被未经授权人员以任何方式查看、修改、复制和删除。
RL-10.1.2-3	合法监听接口相关功能，产品必须提供严格的安全保护机制（如认证、授权、日志记录、加密存储等），防止监听接口被盗用、误用或滥用。
RL-10.1.2-4	日志中不允许记录任何涉及监听对象的敏感信息。
RL-10.1.3-1	在合法监听接口问题定位场景中，严禁采集合法监听原始通信内容。在维护定位时，涉及的合法监听敏感信息（如监听目标、监听时间、通信时间、通信时长等）允许使用，但必须在用户呈现界面(含显示、存储)进行过滤（删除或替换）处理，保证不被人员非法获悉。
RL-10.1.3-2	对具备合法监听接口设备的维护，必须获得现网客户合法监听相关部门的授权，并在其指定人员的监督下进行；对合法监听接口的维护操作只能在现网进行，相关信息不允许带出运营商网络，并在GPI资料中进行上述提示。
RL-10.1.3-3	相关维护操作必须有详细的日志记录。
RL-10.1.3-4	产品CPI资料中对产品涉及合法监听接口的维护功能进行描述和声明：
（1）描述该功能（指接口的维护功能，如维护操作的命令等）的目的、范围、处理方式、时限等。 
（2）声明：要求客户使用该功能时遵从当地适用的法律法规。
RL-10.1.3-5	禁止出于保障网络运营和服务目的提供采集最终用户原始通信内容（语音类、短信/彩信类、传真类、数据业务类）的功能，如录音、抓包，必须通过版本的形式去除（如编译宏、模块化加载，不允许采用License控制）。
RL-10.1.3-6	如果问题定位不得不依赖于通信内容，产品必须提供替代方案（如：使用测试号码进行测试、采集特定音频、语音质量相关指标等方案），确保所采集的内容无法还原为最终用户的原始通信内容，防止相关功能被怀疑为非法监听。
RL-10.1.3-7	对于符合业界标准或惯例的功能（如端口镜像、IP抓包、串口抓包、ACL traffic mirror），且产品本身不对抓取的数据进行解析和处理，允许保留。
RL-10.1.3-8	产品信令跟踪功能中如果涉及短信内容：
必须在呈现界面上（如命令行显示、trace、LMT显示界面、文件存储/导出等）对短信内容进行过滤，以确保在任何情况下任何人无法看到原始的短信内容。
RL-10.1.3-9	除满足上述要求外，对于涉及采集用户通信内容的相关功能还需满足以下要求：
1）必须提供严格的安全控制机制（管理员级的操作权限，日志记录，功能缺省关闭等）；
2）任何涉及客户现网数据的相关操作必须得到客户的授权，并在GPI资料中增加相关提示。
3） 产品CPI资料中对产品涉及采集用户通信内容的功能进行描述和声明：
    - 描述该功能采集用户通信内容的目的、范围、处理方式、时限。
    - 声明：要求客户使用该功能时遵从当地适用的法律法规。
RL-10.1.3-10	对于专用安全类产品（如防火墙、IDS、IPS、AntiDDos、沙箱、大数据安全分析等）提供的内容过滤、URL过滤、行为审计、内容审计、应用行为控制、APT检测、事件关联分析、攻击取证等特性，其主要作用是通过监控、识别最终用户的通信内容，对用户行为进行管控和审计，对网络中可能存在的威胁进行分析和预警，若不违反当地法律要求，且已被业界主流友商安全产品普遍支持，可以采集最终用户原始通信内容（这些特性是否属于敏感特性，依据公司敏感特性标准另行判断）。
RL-10.1.3-11	产品出于维护定位目的，需要在系统崩溃等异常情况下提供自动触发记录系统状态的功能（如黑匣子功能），若无法人为控制且记录的内容也无法确认是否包含通信内容，相关行为符合业界惯例，则不违反红线10.1.3。
RL-10.1.4-1	终端产品禁止提供“在通信双方未知情/未许可的情况下，通过录音、远程抓包等任何方式获取可还原为用户原始通信内容并上传”的功能和特性，必须在代码中彻底删除该类功能和特性，提供干净版本。
RL-10.1.4-2	终端产品是否允许提供通话录音功能参考以下内容：
 - 国内版本允许提供通话录音功能（无需备案）；
 - 海外通用版本默认不允许提供，如果客户有明确的定制需求，需进行法律评估并提交评审。

6、在界面展示原始通信内容时，如果原始通信内容中存在敏感信息，需要对敏感信息进行匿名化处理。

2.1.9.8.1.3.1红线要求

1)使用此类工具前，应事先向客户主动提供该工具的说明；
2)任何维护操作必须得到客户的授权，禁止进行超出客户审批范围的任何操作；
3)将问题定位数据传出客户网络必须得到客户的授权。
RL-11.1.2-1	允许产品正常业务（例如：建立通信连接及计费）的需要，采集、处理、存储个人数据，但必须根据实际安全风险提供必要的安全保护机制（如认证、权限控制、日志记录等），以防止个人数据被泄漏、丢失、破坏。
RL-11.1.2-2	必须在产品CPI（客户资料）中对产品涉及用户隐私的功能进行描述和声明：
- 描述：该功能处理用户个人数据的目的、范围、处理方式、时限
- 声明：要求客户使用该功能时遵从当地适用的法律法规。
RL-11.1.2-3	对于基于现网设备中的个人数据进行进一步整理、分析，且有华为人员参与数据处理的增值服务场景，允许采集、处理系统中的个人数据，但必须遵循如下要求：
1）对个人数据采集或处理的相关操作，必须提供安全保护机制（如认证、权限控制、日志记录等），防止个人数据被泄漏。
2）对于所涉及的个人数据，应支持在呈现界面上（如显示界面、文件存储/导出等）进行过滤或匿名化或假名化。
3）相关操作必须得到客户的授权，禁止进行超出客户审批范围的任何操作，在完成服务后，必须在客户的监督下对所涉及的个人数据进行安全删除，针对敏感市场未经客户授权禁止传出客户网络，并在相关GPI资料中增加提示。
4） 必须在产品CPI资料中对产品涉及用户隐私的功能进行描述和声明：
    - 描述该功能处理用户个人数据的目的、范围、处理方式、时限。
    - 声明：要求客户使用该功能时遵从当地适用的法律法规。
